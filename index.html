    • Le Workflow

    1/ Git init
    2/ Touch .gitignore
    Touch index.html - Touch style.css

    3/Mkdir css - Mkdir js - Mkdir img
    4/ Git status
    5/ Git commit -m"nom du commit"

    6/ Git push origin master
    7 Git pull origin master

    • Git init: Initialize the new project
    • Mkdir: Make a new directory
    • Touch: Create a new file
    • Git Status: Check the status of the project
    • Git Add: Add new directory and files
    • Git commit -m"le nom du commit":
      Save the files and named it
    • Git push -u origin master: Send the files and directory to github repository from Home
    • Git pull origin master: Get back the files from Github repo to Home
    • Git log: View of old commit
    • Git log --oneline: View the commit in one line
    • Git log --oneline -p (nom du fichier): View of the commit of the called file
    • Git diff: View of the last changes
    • Git config global color-uni:< Add color
    • Git config --list: View of the state of the config
    • Git checkout "ID" "nom du fichier":Get back to the old version of the file identifie with an ID
    • Git branch: View of the branch
    • Git branch checkout "nom de la branch": Get change to the new branch
